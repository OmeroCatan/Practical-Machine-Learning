Provided Background
-------------------

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement â€“ a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, your goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: <http://web.archive.org/web/20161224072740/http:/groupware.les.inf.puc-rio.br/har> (see the section on the Weight Lifting Exercise Dataset).

Project Goal
------------

The goal of the project is to predict the manner in which the exercise was conducted. This is defined as the "classe" variable in the training set data. Once a model is built, it will be used to predict 20 different test cases.

Clean Data
----------

The clean data contains 53 of the original 160 variables. There are multiple methods to use when creating a predictive model. Three include CLassification Trees with K-Fold Cross Validation, Random Forests, and Boosting.

When using Classification Trees with K-Fold Cross Validation, the larger the number of folds ("K"), the less bias and more variance you will experience. With Random Forests, at each split you bootstrap the variables to grow multiple trees and vote. With Boosting, the goal is to calculate weights of variables based upon errors.

My original plan was to test the three methods above and select the method with the highest accuracy to run the test cases on. However, I kept comming across computer memory limitations.

In this project, we are attempting to accuratly predict the manner in which the exercise was conducted. Accuracy is a proficency of the Random Forests method so I selected that method to base the model on.

Random Forests Output
---------------------

In random forests, the unbiased estimate of the test set error is estimated internally during the run of the model. Therefore, cross-validation is not required to be ran separately.

The model created has an accuracy of 99.41%. Therefore, my predcted accuracy for the out-of-sample error is 0.59%. This is an acceptable accuracy percentage to move forward and apply to the test cases.
